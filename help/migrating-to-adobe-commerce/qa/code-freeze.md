# Adobe Commerceへの移行 – コードのフリーズ

{{only-for-on-prem-commerce-cloud}}

Adobe Commerceの移行中にコードのフリーズを実装することは、安定してローンチを成功させるために非常に重要です。 新しいコードのデプロイメントを停止することで、QA チーム、開発者、関係者が静的なコードベースを使用して作業できるようになり、信頼性が向上し、欠陥のリスクが軽減されます。 このプロセスでは、徹底的なテストと検証、優先度の高いバグへの対処、サイトのあらゆる側面の期待通りの機能の確保が必要です。 適切に実行されたコードの凍結は、プロジェクト全体にメリットをもたらし、直前の問題を防ぎ、Adobe Commerceへのスムーズな移行を実現します。


## この動画は誰のためのものでしょうか？

* プロジェクトマネージャーは、プロジェクトをスムーズに遂行し、リスクを最小限に抑えようとする必要があります。
* 最終的なテストのための安定したコードベースの必要性を理解しようとしている開発者。
* テストのタイムラインと期待値を明確にしようとしているところです。
* 最終検証とローンチの準備状況の重要性を認識したいビジネス関係者。

## ビデオコンテンツ {#video-content}

* 最終QA テストと検証のための静的なコードベースを確保して、直前の問題を防ぐ。
* 手動および自動化されたテスト、読み込みテスト、侵入テストを調整し、サイトのあらゆる側面が期待どおりに機能していることを確認します。
* ビジネス関係者、開発者、IT チームを検証プロセスに関与させ、ローンチをスムーズに成功させることができます。
