# Adobe コンポーザブルカタログデータモデルのポリシー

**カタログビュー**&#x200B;が、買い物客が統合されたベースカタログから見るものを形成するレンズである場合、**ポリシー**&#x200B;はそのレンズで構成されています。 このチュートリアルでは、ポリシーの概要、**Carvelo Automobiles**&#x200B;のデモンストレーション シナリオで&#x200B;**STATIC**&#x200B;と&#x200B;**トリガー**&#x200B;のポリシーがどのように連携するのか、また、カタログを再構築せずにポリシーの更新がすぐに有効になる理由について説明します。

## この動画は誰のためのものでしょうか？

* Adobe Commerce Optimizerでカタログビューとマーチャンダイジングルールを設定するCommerceソリューションアーキテクトおよび開発者

## ビデオコンテンツ

* 製品属性に対するデータアクセスフィルターとしてのポリシー
* 複数のポリシーをCelport カタログビューで組み合わせる方法（ブランドとパーツ カテゴリ）
* 永続的なビジネスルールの静的ポリシー
* API リクエストヘッダーによってアクティブ化されたトリガーポリシー（例：`AC-Policy-Brand`）
* カタログを再構築することなく、日常業務でポリシーを更新する

>[!VIDEO](https://video.tv.adobe.com/v/3491413?learn=on)

**ポリシー**&#x200B;は&#x200B;**データアクセスフィルター**&#x200B;です。 製品属性を検査し、カタログビューで表示される製品を決定するルールを適用します。 ポリシーは、共有されたコンポーザブルカタログの上に配置され、カタログデータを複製しません。

## 静的ポリシー

**STATIC** ポリシーは&#x200B;**常に**&#x200B;にあります。 買い物客の行動やセッションの状態に関係なく、永続的なビジネスルールを適用できます。

Celport シナリオでは、静的ルールには次のものが含まれます。

* Celportは、**Bolt**&#x200B;および&#x200B;**Cruz**&#x200B;のブランドのみを販売します。
* セルポートには&#x200B;**ブレーキ**&#x200B;と&#x200B;**サスペンション**&#x200B;のパーツのみが表示されます。

これらのルールは決してオフにはなりません。 静的ポリシーは、**ライセンス契約**、**領域制限**、**ブランド権限**&#x200B;が有効な場所です。 一度設定すれば、Adobe Commerce Optimizerが毎回リクエストに応じて自動的に適用します。

## トリガーポリシー

値ソースが&#x200B;**トリガー**&#x200B;のポリシーは、**排他的なポリシー**&#x200B;と呼ばれることもあります。 カタログ ビューでは、API呼び出しのヘッダーにトリガーが指定された&#x200B;**場合にのみ、そのポリシー**&#x200B;が実行されます。

例えば、Experience Delivery Services （EDS）ストアフロントでは、**All Brands**、**Aurora**、**Bolt**、**Cruz**&#x200B;などのドロップダウンを提供できます。

* 最初のページビューでは何も選択されないため、追加のブランドヘッダーは送信されません。
* 買い物客が&#x200B;**Bolt**&#x200B;を選択すると、基になるAPIによって`AC-Policy-Brand` ヘッダーが`Bolt`に設定されます。
* 買い物客が&#x200B;**Cruz**&#x200B;を選択すると、同じヘッダーが`Cruz`に設定されます。

カタログビューでは、そのヘッダーが存在する場合にのみトリガーポリシーが適用されます。これにより、永続的な静的ルールを変更することなく、インタラクティブなフィルタリングがサポートされます。

## 静的データとトリガーの連携

**STATIC** ポリシーは、永続的なルールを処理します。 **トリガー** ポリシーは、インタラクティブなポリシーを処理します。 単一のカタログビューで一緒に使用すると、**コンプライアンス**&#x200B;と&#x200B;**柔軟性**&#x200B;の両方が提供されます。これは、固定の品揃え境界に加えて、買い物客による絞り込みです。

## カタログを再構築せずにポリシーを更新する

日々の業務において、すばやくポリシーを変更できます。 セルポートのライセンス契約で、ブレーキやサスペンションと同様に&#x200B;**タイヤ**&#x200B;が許可されているとします。

1. 関連するポリシーを開きます。
1. 値リストに&#x200B;**タイヤ**&#x200B;を追加します。
1. 保存。

その変更は&#x200B;**すぐに反映されます**。 カタログの再構築、再デプロイメント、待機時間はありません。 Celportのストアフロントへの次のリクエストには、更新が反映されます。

ポリシーは、**共有カタログ**&#x200B;の軽量フィルターであり、個別のカタログコピーに組み込まれたルールではありません。 **データではなくルールを変更します。**
