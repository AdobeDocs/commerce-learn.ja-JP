# カート価格ルールの作成

カートの価格規則ショッピングカート内の商品に、設定した条件に基づく割引を適用します。 割引は、条件が満たされた場合、または顧客が有効なクーポンコードを入力した場合に、自動的に適用されます。 小計の下のカートに割引が表示されます。 ステータスと日付範囲を変更することで、シーズンまたはプロモーションのルールをオンまたはオフにできます。

## 価格表示の問題

場合によっては、各行項目に適用される割引を表示する必要がありますが、表示される値が正確に一致しない場合があります。 これは、カート価格ルールが複数の商品に1つの割引を適用し、分割が均等に2つの小数点以下桁に分割されない場合に発生します。

>[!BEGINSHADEBOX]

カート価格ルール = カート内の2つの商品に適用される10%割引
価格ルールを有効にする条件：買い物かごの合計品目は2です
アクションは製品価格割引のパーセントを適用し、その割引額は10です

2つの商品がカートに追加され、それぞれ19.95 ドルです

割引額を計算するには、商品価格に0.1を掛けます

19.95 x 0.1 = 1.995

これは問題です。小数点以下桁が2桁ではなく3桁です。 ドルに変換することは今や問題です

>[!ENDSHADEBOX]

### 解決策

管理画面で販売者の場合、最も明確なアプローチは、各注文行に割引が表示されている金額を表示することです。 注文合計を正確に保つには、最初の行項目を上に丸め、残りの行項目に3番目の小数をドロップします。 このシナリオを確認：

>[!BEGINSHADEBOX]

上記の買い物かごルールと同じ10%割引
買い物かごに2個の商品を追加します（19.95）

各商品は1.995 ドルの割引を受けることができます
商品1 - 19.95 x 0.1 = 1.995
2 - 19.95 x 0.1 = 1.995

合計3.99は、お客様に割引として提供されます

管理画面でストアオーナーに行アイテムを表示する際に，
最初の項目を調整し、2.000まで丸める必要があります。 2番目の項目の場合は、3番目の小数点をドロップします。
商品1 = 2.00
商品2 = 1.99

合計された2つの製品の合計割引は、顧客に提供された実際の割引と一致します。
>[!ENDSHADEBOX]

以下は、このシナリオを含む注文の管理画面に表示されるスクリーンショットです。

異なる値を持つ順序付きアイテムを表示する![管理者ビュー](../assets/commerce-admin-cart-price-rule-values-different.png)

### その他の潜在的なソリューションとその理由

>[!BEGINSHADEBOX]

上記の買い物かごルールと同じ10%割引
買い物かごに2個の商品を追加します（19.95）

各商品は1.995 ドルの割引を受け、
しかし、それらを切り上げると、割引が多すぎることがわかります。

商品1 - 19.95 x 0.1 = 1.995
製品2 - 19.95 x 0.1 = 1.995

すべての項目を切り上げる場合に変換
製品1新規値は2.00
製品2新規値は2.00

実際に、合計3.99が顧客に割引として提供され
しかし、我々が切り上げれば、それは400 ドルが与えられたことを示し、それは間違っている。

2.00 + 2.00 = $4.00

>[!ENDSHADEBOX]

同様の問題3番目の小数点がすべての項目に対して削除された場合、提供される割引が少なすぎることになります。

>[!BEGINSHADEBOX]

上記の買い物かごルールと同じ10%割引
買い物かごに2個の商品を追加します（19.95）

各商品には1.995 ドルの割引が適用されますが、3番目の10進数を下げるだけにすると、次のようになります。
商品1 - 19.95 x 0.1 = 1.995
製品2 - 19.95 x 0.1 = 1.995

すべての項目の3番目の小数をドロップするように変換
製品1新しい値は1.99
製品2新しい値は1.99

実際に、合計3.99が顧客に割引として提供され
しかし、3番目の10進数を落とすと、3.98 ドルが与えられたことが示され、それは間違っています。

1.99 + 1.99 = $3.98

>[!ENDSHADEBOX]
